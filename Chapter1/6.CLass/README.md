## Class

### Class (類別)
在Java中，Class（類別）是一個重要的概念，它用於定義對象的屬性和行為。Class是Java面向對象編程的基礎，它描述了對象的結構和行為，並作為創建對象的模板。      
      
一個Class可以包含以下元素：      
      
1. 屬性（Fields）：Class中的屬性是對象的狀態信息。它們描述了對象具有的數據特徵，例如顏色、大小等。屬性可以是各種數據類型，如整數、字符串、布爾值等。

2. 方法（Methods）：Class中的方法是對象的行為。它們定義了對象能夠執行的操作，例如計算、修改屬性值、返回結果等。方法可以接受參數，並且可以有返回值或是沒有返回值。

3. 構造方法（Constructor）：構造方法是一種特殊的方法，用於創建和初始化對象。它在創建對象時被調用，並且可以接受參數來初始化對象的屬性。

4. 內部類（Inner Class）：Class中還可以包含其他Class，這些被稱為內部類。內部類可以訪問外部類的屬性和方法，提供了更強大的封裝和組織代碼的能力。
      
通常，我們使用Java的Class來創建對象。通過實例化（使用關鍵字new），我們可以根據Class的定義創建具體的對象。對象是Class的實例，它具有Class中定義的屬性和方法。      
      
總的來說，Java的Class提供了一種組織代碼的方式，它封裝了對象的屬性和行為，並提供了創建和操作對象的能力。Class在Java中是面向對象編程的基石，幫助我們建立可重用和結構化的代碼。      

### Constructor (構造方法)
在Java中，構造方法（Constructor）是一種特殊的方法，用於創建和初始化對象。它與類別同名並且沒有返回值，主要用於在創建對象時進行必要的初始化操作。      
      
構造方法具有以下特點：      
      
1. 方法名與類名相同：構造方法的名稱必須與其所屬類別的名稱完全相同，包括大小寫。這樣編譯器才能識別構造方法。      
      
2. 沒有返回值：構造方法不像普通方法一樣有返回值，包括void。當使用new關鍵字創建對象時，構造方法會自動被調用，並且它的任務是初始化對象。
      
3. 可以擁有參數：構造方法可以接受參數，這些參數用於初始化對象的屬性。通過在構造方法的參數列表中聲明參數，我們可以在創建對象時向構造方法傳遞初始化數值。
      
4. 可以有多個重載版本：一個類別可以擁有多個構造方法，只要它們的參數列表不同。這種稱為方法重載（Method Overloading）的機制使得我們可以根據不同的需求來初始化對象。
      
構造方法在Java中扮演著非常重要的角色，它確保了對象在創建時被正確地初始化。通常，我們在類別中定義一個或多個構造方法，根據需求選擇合適的構造方法來創建對象。

### public 與 private 的差異
以下是`public`和`private`的差別：         
         
1. `public`：當成員被聲明為`public`時，它可以在任何地方被訪問，包括同一個類、不同的類、不同的包（package）和不同的模塊（module）。其他類和模塊可以直接訪問和使用`public`成員。
         
2. `private`：當成員被聲明為`private`時，它只能在同一個類內部被訪問。其他類無法直接訪問`private`成員。這種訪問限制確保了對象的封裝性和信息隱藏性，使得類的內部細節對外部不可見。
         
下面是一個示例，演示了`public`和`private`的使用：         

```java
public class MyClass {
    public int publicVariable; // public屬性
    private int privateVariable; // private屬性

    public void publicMethod() {
        // public方法
        // 可以在任何地方訪問和調用
    }

    private void privateMethod() {
        // private方法
        // 只能在同一個類內部訪問和調用
    }
}
```
         
### fields and local variable
在Java中，有兩種主要的變量類型：字段（Fields）和局部變量（Local Variables）。它們在作用域、生命周期和訪問性方面有所不同。         
         
1. 字段（Fields）：
   - 字段是類或對象中聲明的變量，用於存儲對象的狀態或屬性。
   - 字段可以是實例字段（Instance Fields）或靜態字段（Static Fields）。
   - 實例字段屬於對象的一部分，每個對象都有自己的一份拷貝。它們可以通過對象訪問。
   - 靜態字段屬於類本身，只有一份拷貝，被所有對象共享。它們可以通過類名訪問。
         
2. 局部變量（Local Variables）：
   - 局部變量是在方法、代碼塊或構造函數內部聲明的變量。
   - 它們只在聲明它們的方法、代碼塊或構造函數中可見，並且在它們的作用域內有效。
   - 局部變量必須初始化後才能使用，並且在離開作用域後就被銷毀。
   - 局部變量不具有訪問修飾符，並且只能在聲明它們的區域內部訪問。
         
以下是一個示例，演示了字段和局部變量的使用：         
         
```java
public class Example {
    // 實例字段
    private int instanceField;

    // 靜態字段
    private static int staticField;

    public void method() {
        // 局部變量
        int localVar = 10;

        // 可以訪問實例字段和局部變量
        instanceField = 20;
        System.out.println("Local variable: " + localVar);
    }

    public static void main(String[] args) {
        Example example = new Example();
        example.method();

        // 可以訪問靜態字段
        staticField = 30;
        System.out.println("Static field: " + staticField);
    }
}
```

### This keyWord
在Java中，`this`是一個關鍵字，代表了當前對象的引用。它可以用來引用當前對象的成員（字段和方法），並區分當前對象的成員與參數或局部變量。                  
         
以下是`this`關鍵字的用法和特點：         
         
1. 引用當前對象：
   - `this`關鍵字用於引用當前對象本身。
   - 它可以在類的內部的方法、構造函數或代碼塊中使用。
         
2. 區分成員和參數：
   - 當成員名稱與參數或局部變量名稱相同時，可以使用`this`關鍵字來區分它們。
   - `this`關鍵字可以將成員與參數或局部變量進行區分，確保準確訪問到成員。
         
3. 調用其他構造函數：
   - `this`關鍵字還可以在構造函數中使用，用於調用同一類中的其他構造函數。
   - 這種用法稱為"構造函數鏈"，可以在一個構造函數中調用同一類中的其他構造函數，方便對對象進行初始化。
         
下面是一個示例，演示了`this`關鍵字的使用：         

```java
public class MyClass {
    private int value;

    public MyClass(int value) {
        this.value = value; // 使用this區分成員和參數
    }

    public void setValue(int value) {
        this.value = value; // 使用this區分成員和局部變量
    }

    public int getValue() {
        return this.value; // 使用this引用成員變量
    }

    public MyClass() {
        this(0); // 使用this調用另一個構造函數
    }
}
```

### static 
在Java中，靜態屬性（Static Fields）和靜態方法（Static Methods）是與類關聯而不是與對象關聯的成員。它們屬於類本身，而不是類的實例。靜態成員在整個類中是共享的，所有對象都可以訪問和修改它們。         
         
以下是靜態屬性和靜態方法的一些特點：         
         
1. 靜態屬性：靜態屬性是用 `static` 修飾的類成員變量。它們在內存中只有一份拷貝，被所有對象共享。可以直接通過類名訪問靜態屬性，而不需要創建類的對象。
         
2. 靜態方法：靜態方法是用 `static` 修飾的類成員函數。它們不需要通過對象來調用，可以直接通過類名訪問。靜態方法不能訪問非靜態成員，因為它們不屬於具體的對象。
         
3. 共享特性：靜態成員在所有對象之間共享，可以用於保存和訪問共享的數據或提供共享的功能。由於靜態成員不依賴於具體的對象，它們在程序啟動時就已經存在，可以在沒有對象的情況下訪問。

### Modifier (修飾符)
在Java中，修飾符（Modifier）用於修改類、字段、方法、構造函數和其他程序元素的行為和屬性。Java提供了多個修飾符，每個修飾符都有特定的含義和作用。

以下是常用的Java修飾符：

1. 訪問修飾符（Access Modifiers）：
   - `public`：可以在任何地方訪問，沒有訪問限制。
   - `private`：只能在同一類內訪問，對其他類不可見。
   - `protected`：可以在同一類、同一包內以及不同包中的子類中訪問。
   - 默認（沒有顯式修飾符）：可以在同一包內訪問，對其他包中的類不可見。

2. 非訪問修飾符：
   - `final`：表示最終的、不可修改的特性。對於類，表示不能被繼承；對於方法，表示不能被重寫；對於變量，表示不能重新賦值。
   - `static`：表示靜態的，屬於類本身而不是類的實例。靜態字段和方法可以通過類名直接訪問。
   - `abstract`：只能用於抽象類和抽象方法，表示該類或方法沒有具體實現，需要被子類實現。
   - `synchronized`：用於多線程環境，表示該方法或代碼塊在同一時間只能被一個線程執行。
   - `volatile`：用於多線程環境，表示該變量的值在不同線程中是可見的，並且不會被緩存。

3. 其他修飾符：
   - `final`：用於方法參數，表示參數的值不能被修改。
   - `native`：表示該方法使用本地代碼實現，通常與JNI（Java Native Interface）一起使用。
   - `strictfp`：用於類、接口和方法，表示使用精確的浮點計算，確保在不同平台上的一致性。
   - `transient`：用於字段，表示該字段不會被序列化。
   - `default`（Java 8+）：用於接口中的方法，表示該方法有默認實現。

這些修飾符可以組合使用，並根據需要對類、方法、字段等進行修飾。
